# Infinity ComfyUI Agent Notes

## 项目协作规则

- 默认使用中文沟通，保持说明直接、准确、可验证。
- 修改代码前先理解现有结构和约定，优先沿用项目内已有模式。
- 不要回滚用户或其他工具产生的无关改动；遇到冲突先确认影响范围。

## 每次修改后的固定收尾

每次完成任何项目文件修改后，都必须执行完整收尾流程：

1. 运行构建验证，至少执行 `npm run build`。
2. 确认 `git status --short` 中只包含本次相关改动。
3. 提交本次改动。
4. 创建新的递增版本 tag。
5. push 当前分支和新 tag。

如果 build、commit、tag 或 push 任一步失败，最终回复必须明确说明失败命令、原因和当前仓库状态。

## Tag 版本规则

tag 固定使用 `v大版本.小版本.迭代号` 格式，例如 `v1.2.3`。

- 修 bug：只增加迭代号，例如 `v1.2.3` -> `v1.2.4`。
- 新增或修改功能：增加小版本号，并将迭代号归零，例如 `v1.2.3` -> `v1.3.0`。
- 页面重构：增加大版本号，并将小版本号与迭代号归零，例如 `v1.2.3` -> `v2.0.0`。
